# Credits

The refactoring names, code-smell names, and the step-by-step mechanics in this skill are
derived from the established refactoring literature — principally:

- Martin Fowler, *Refactoring: Improving the Design of Existing Code*, 2nd edition
  (Addison-Wesley, 2019). The catalog structure (smells, named refactorings, and the
  small-step "mechanics" format) originates there.
- The legacy-code testing techniques referenced in `09-getting-code-under-test.md` draw on
  the broader body of work on working with untested code (e.g., Michael Feathers,
  *Working Effectively with Legacy Code*).

The mechanics in this skill were rewritten in original wording and reorganized for use by
an AI coding agent; the refactoring and smell *names* are common industry vocabulary. This
attribution is kept here, separate from the agent-facing skill text, so the skill's
narration stays free of name-dropping while the provenance remains clearly acknowledged.

If you distribute or modify this skill, please keep this attribution intact.
